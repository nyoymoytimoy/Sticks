# 998 — CLAUDE.md / AGENTS.md Relocation Fix

**Status**: Implemented
**Date**: 2026-07-15

> **Note on numbering**: like `999-monorepo-restructure.md`, this is an
> infrastructure/chore fix, not a numbered feature spec — it doesn't shift
> the 011+ workflow sequence.

## Problem / Context

`CLAUDE.md` and `AGENTS.md` are Next.js-specific files auto-generated by
`create-next-app` during spec 001 (scaffolding) — `AGENTS.md` warns that
this project runs Next.js 16 with breaking changes and to check
`node_modules/next/dist/docs/` before touching Next.js APIs; `CLAUDE.md` is
a one-line `@AGENTS.md` include.

When spec 999 restructured the repo into `frontend/` + `backend/`
workspaces, the entire Next.js project — including these two files — moved
into `frontend/`. That was wrong for two reasons, both caught by the user
noticing the file's location while reviewing spec 999's changes:

1. Claude Code only reads `CLAUDE.md` from a session's working-directory
   root. A session opened at the repo root (`ticked/`, the default) never
   saw this file at all — only a session opened specifically inside
   `frontend/` would.
2. `AGENTS.md`'s own file reference, `node_modules/next/dist/docs/`, became
   inaccurate once it moved: npm workspaces hoist `node_modules` to the
   repo root, not into `frontend/`, so the path only resolves correctly
   from the root.

## Scope

**In scope**: `git mv frontend/CLAUDE.md CLAUDE.md` and
`git mv frontend/AGENTS.md AGENTS.md` — pure relocation, no content changes
needed, since neither file contained anything frontend-specific beyond the
now-corrected `node_modules` path reference.

**Deliberately out of scope**: no change to the files' content — moving
them to the root is a complete fix on its own; nothing else needed
updating.

## Acceptance criteria

- [x] `CLAUDE.md` and `AGENTS.md` exist at the repo root, not
      `frontend/`.
- [x] `node_modules/next/dist/docs/` (referenced in `AGENTS.md`) resolves
      correctly relative to the repo root, verified directly (`ls
      node_modules/next/dist/docs/` from root lists the expected Next.js
      doc folders).

## Test plan

Not applicable — file relocation with no behavioral surface to test via
Playwright.
